Introduction to Python

What is Python?

Python is a high-level, interpreted programming language known for its simplicity and readability. It was created by Guido van Rossum and first released in 1991. Python is widely used for web development, data analysis, artificial intelligence, scientific computing, automation, and more.

Key Features of Python:

  • Simple Syntax: Python's syntax is designed to be intuitive and easy to read, making it accessible to beginners.
  • Interpreted Language: Python code is executed line by line, which means you don't need to compile your code before running it.
  • Dynamic Typing: Python automatically determines the data type of a variable at runtime, so you don't need to declare types explicitly.
  • Extensive Standard Library: Python comes with a vast standard library that provides tools and modules for various tasks, from file handling to web development.

Installing Python

To get started with Python, you need to install it on your system. Here's how to do it:

Download Python:

  • Visit the official Python website.
  • Download the latest version of Python for your operating system (Windows, macOS, or Linux).

Install Python:

  • Windows: Run the downloaded executable file and follow the installation prompts. Ensure that you check the box that says "Add Python to PATH."
  • macOS/Linux: Python is usually pre-installed on these systems. You can check the installed version by running python3 --version in the terminal. If it's not installed, you can use a package manager like brew (macOS) or apt-get (Linux) to install it.

Verify Installation:

python --version

This should display the installed Python version, confirming that Python is correctly installed.

Python IDEs (e.g., PyCharm, VS Code, Jupyter)

An Integrated Development Environment (IDE) provides tools for writing, testing, and debugging code. Here are some popular Python IDEs:

  • PyCharm: PyCharm is a full-featured IDE specifically designed for Python. It offers code analysis, a graphical debugger, and an integrated terminal. Usage: Best suited for large projects with complex codebases.
  • VS Code (Visual Studio Code): VS Code is a lightweight, open-source code editor with Python support through extensions. It offers features like IntelliSense, debugging, and Git integration. Usage: Ideal for small to medium-sized projects and versatile across different programming languages.
  • Jupyter Notebook: Jupyter Notebook is an open-source web application that allows you to create and share documents containing live code, equations, visualizations, and narrative text. Usage: Primarily used in data science and machine learning for interactive coding and data analysis.

Python Basics

Variables and Data Types

Variables are containers for storing data values. Python automatically assigns the data type based on the value assigned to the variable.

a. Integers

Definition: Whole numbers, positive or negative, without decimals.

Example:

python
x = 10
y = -5
print(x)  # Output: 10
print(y)  # Output: -5

Use: Used for counting, indexing, or any operation where only whole numbers are needed.

b. Floats

Definition: Numbers with decimals.

Example:

python
pi = 3.14
radius = 5.0
print(pi)     # Output: 3.14
print(radius) # Output: 5.0

Use: Used in mathematical calculations that require precision, such as scientific computations.

c. Strings

Definition: A sequence of characters enclosed within single (') or double (") quotes.

Example:

python
name = "Sandeep"
university = 'Sandip University'
print(name)         # Output: Sandeep
print(university)   # Output: Sandip University

Use: Used for storing and manipulating textual data.

d. Booleans

Definition: Represents one of two values: True or False.

Example:

python
is_student = True
is_graduated = False
print(is_student)   # Output: True
print(is_graduated) # Output: False

Use: Used for decision-making in conditional statements.

e. Lists

Definition: An ordered collection of items, which can be of mixed data types. Lists are mutable (i.e., items can be added, removed, or changed).

Example:

python
fruits = ["apple", "banana", "cherry"]
numbers = [1, 2, 3, 4, 5]
mixed_list = [1, "apple", True]
print(fruits)       # Output: ['apple', 'banana', 'cherry']
print(numbers)      # Output: [1, 2, 3, 4, 5]
print(mixed_list)   # Output: [1, 'apple', True]

Use: Used to store multiple items in a single variable, especially when order matters.

f. Tuples

Definition: An ordered collection of items, similar to lists, but immutable (i.e., items cannot be changed).

Example:

python
coordinates = (10, 20)
person = ("Sandeep", 25, "Student")
print(coordinates)  # Output: (10, 20)
print(person)       # Output: ('Sandeep', 25, 'Student')

Use: Used when a collection of items should not be changed.

g. Sets

Definition: An unordered collection of unique items.

Example:

python
colors = {"red", "green", "blue"}
numbers = {1, 2, 3, 4, 5}
print(colors)  # Output: {'red', 'green', 'blue'}
print(numbers) # Output: {1, 2, 3, 4, 5}

Use: Used to store items where uniqueness is important.

h. Dictionaries

Definition: A collection of key-value pairs, where each key is unique.

Example:

python
student = {"name": "Sandeep", "age": 25, "university": "Sandip University"}
print(student["name"])  # Output: Sandeep
print(student["age"])   # Output: 25

Use: Used to store data that is associated with a unique key.

Operators

Operators are symbols used to perform operations on variables and values.

a. Arithmetic Operators

Description: Perform basic mathematical operations.

Examples:

python
x = 10
y = 3
print(x + y)  # Output: 13 (Addition)
print(x - y)  # Output: 7  (Subtraction)
print(x * y)  # Output: 30 (Multiplication)
print(x / y)  # Output: 3.3333333333333335 (Division)
print(x % y)  # Output: 1  (Modulus)
print(x ** y) # Output: 1000 (Exponentiation)
print(x // y) # Output: 3 (Floor Division)

b. Comparison Operators

Description: Compare two values and return a Boolean (True or False).

Examples:

python
a = 5
b = 10
print(a == b)  # Output: False (Equal)
print(a != b)  # Output: True  (Not equal)
print(a > b)   # Output: False (Greater than)
print(a < b)   # Output: True  (Less than)
print(a >= b)  # Output: False (Greater than or equal to)
print(a <= b)  # Output: True  (Less than or equal to)

c. Logical Operators

Description: Used to combine conditional statements.

Examples:

python
x = True
y = False
print(x and y)  # Output: False (Logical AND)
print(x or y)   # Output: True  (Logical OR)
print(not x)    # Output: False (Logical NOT)

d. Assignment Operators

Description: Assign values to variables.

Examples:

python
x = 5       # Assigns value 5 to x
x += 3      # x = x + 3, Output: 8
x -= 2      # x = x - 2, Output: 6
x *= 4      # x = x * 4, Output: 24
x /= 2      # x = x / 2, Output: 12.0
x %= 5      # x = x % 5, Output: 2.0
x **= 3     # x = x ** 3, Output: 8.0
x //= 2     # x = x // 2, Output: 4.0

e. Bitwise Operators

Description: Perform operations on binary digits.

Examples:

python
a = 5  # 0101 in binary
b = 3  # 0011 in binary
print(a & b)  # Output: 1  (Bitwise AND)
print(a | b)  # Output: 7  (Bitwise OR)
print(a ^ b)  # Output: 6  (Bitwise XOR)
print(~a)     # Output: -6 (Bitwise NOT)
print(a << 1) # Output: 10 (Bitwise Left Shift)
print(a >> 1) # Output: 2  (Bitwise Right Shift)

Input and Output

a. print() Function

Description: Outputs data to the console.

Example:

python

Use: Used to display information to the user.

b. Reading Input from the User with input()

Description: Takes user input as a string.

Example:

python

Use: Used to collect input from the user.

Python Concepts: Conditional Statements, Loops, and List Comprehensions

1. Conditional Statements

Conditional statements are used to execute different blocks of code based on certain conditions. Python provides three main types of conditional statements: if, elif, and else.

if Statement

Used to test a condition. If the condition is True, the block of code inside the if statement is executed.

elif Statement

Stands for "else if" and is used to test multiple conditions. It follows an if statement and allows you to check additional conditions if the initial if condition is False.

else Statement

Used to execute a block of code if none of the previous conditions are True.

Example:


# Example of if, elif, else
x = 10

if x > 15:
print("x is greater than 15")
elif x == 10:
print("x is equal to 10")
else:
print("x is less than 15 and not equal to 10")

Output:


x is equal to 10

Explanation:

The if condition (x > 15) is False, so it skips to the elif statement. The elif condition (x == 10) is True, so it executes the code inside this block and prints "x is equal to 10". The else block is not executed because the elif condition was met.

2. Loops

Loops are used to execute a block of code repeatedly based on a condition. Python provides two main types of loops: for loops and while loops.

for Loops

Used to iterate over a sequence (e.g., list, tuple, dictionary, set, or string) or other iterable objects.

Example:


# Example of for loop
for i in range(5):
print(i)

Output:


0
1
2
3
4

Explanation:

range(5) generates a sequence of numbers from 0 to 4. The for loop iterates over this sequence, and each number is printed on a new line.

while Loops

Repeats a block of code as long as the given condition is True.

Example:


# Example of while loop
count = 0
while count < 5:
print(count)
count += 1

Output:


0
1
2
3
4

Explanation:

The while loop continues to execute as long as count is less than 5. The count variable is incremented by 1 in each iteration, and the current value of count is printed.

3. Loop Control

Loop control statements allow you to alter the behavior of loops. Python provides three loop control statements: break, continue, and pass.

break Statement

Exits the loop immediately and skips the remaining iterations.

Example:


# Example of break
for i in range(10):
if i == 5:
    break
print(i)

Output:


0
1
2
3
4

Explanation:

When i equals 5, the break statement is executed, which terminates the loop. Numbers 0 to 4 are printed, and the loop exits before printing 5.

continue Statement

Skips the rest of the code inside the loop for the current iteration and proceeds to the next iteration.

Example:


# Example of continue
for i in range(10):
if i % 2 == 0:
    continue
print(i)

Output:


1
3
5
7
9

Explanation:

The continue statement is executed when i is even, causing the loop to skip the print(i) statement for even numbers. Only odd numbers are printed.

pass Statement

A placeholder that does nothing and is used where syntactically some code is required but no action is needed.

Example:


# Example of pass
for i in range(5):
if i == 3:
    pass  # Placeholder for future code
else:
    print(i)

Output:


0
1
2
4

Explanation:

When i is 3, the pass statement is executed, which does nothing. For all other values of i, the number is printed.

4. List Comprehensions

List comprehensions provide a concise way to create lists. They consist of an expression followed by a for clause, and optionally, if clauses.

Example:


# Example of list comprehension
squares = [x**2 for x in range(10)]
print(squares)

Output:


[0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

Explanation:

[x**2 for x in range(10)] creates a list of squares of numbers from 0 to 9. The expression x**2 calculates the square of each number, and for x in range(10) iterates through the numbers from 0 to 9.

Example with Conditional:


# Example of list comprehension with condition
even_squares = [x**2 for x in range(10) if x % 2 == 0]
print(even_squares)

Output:


[0, 4, 16, 36, 64]

Explanation:

[x**2 for x in range(10) if x % 2 == 0] creates a list of squares of even numbers from 0 to 9. The condition if x % 2 == 0 ensures that only even numbers are included in the list.

Defining Functions

In Python, a function is defined using the def keyword followed by the function name and parentheses. The function body is indented and contains the code to be executed.

Syntax:

def function_name(parameters):
    # Function body
    return value

Example:

def greet(name):
    return f"Hello, {name}!"

Usage:

print(greet("Sandeep"))  # Output: Hello, Sandeep!

Parameters and Arguments

Parameters are variables listed inside the parentheses in the function definition. Arguments are the values passed to the function when it is called.

Example:

def add(a, b):
    return a + b

result = add(5, 3)  # Arguments: 5, 3
print(result)  # Output: 8

Return Values

The return statement is used to exit a function and return a value to the caller. If no return statement is used, the function returns None by default.

Example:

def square(x):
    return x * x

print(square(4))  # Output: 16

Lambda Functions

Lambda functions are anonymous functions defined using the lambda keyword. They can have any number of arguments but only one expression. The result of the expression is returned.

Syntax:

lambda arguments: expression

Example:

add = lambda x, y: x + y
print(add(5, 3))  # Output: 8

Scope and Lifetime of Variables

Local Variables: Defined inside a function and can only be accessed within that function.

Global Variables: Defined outside all functions and can be accessed from anywhere in the code.

Example:

x = 10  # Global variable

def func():
    y = 5  # Local variable
    print(x)  # Accessing global variable
    print(y)  # Accessing local variable

func()
print(x)  # Output: 10
# print(y)  # This would raise an error because y is local to func

Importing Modules

Python modules are files containing Python code. Modules can be imported into other modules or scripts using the import statement.

Standard Library Modules: These are built-in modules provided by Python, like math, datetime, os, etc.

Example:

import math

print(math.sqrt(16))  # Output: 4.0

Third-party Modules: These are external modules not included in the standard library. They need to be installed using pip.

Example:

# Install with: pip install requests
import requests

response = requests.get('https://api.github.com')
print(response.status_code)  # Output: 200

Creating and Using Custom Modules

You can create your own modules by saving functions and variables in a .py file. You can then import this file into other scripts.

Create a file named mymodule.py:

def multiply(a, b):
    return a * b

def divide(a, b):
    return a / b

Import and use the custom module:

import mymodule

print(mymodule.multiply(4, 5))  # Output: 20
print(mymodule.divide(20, 4))  # Output: 5.0

Explanation

  • Defining Functions: Helps in breaking down tasks into smaller, manageable pieces, making code reusable and modular.
  • Parameters and Arguments: Allow functions to operate on different inputs, making them more flexible.
  • Return Values: Enable functions to send back results, which can be used elsewhere in the code.
  • Lambda Functions: Provide a concise way to write small, throwaway functions, useful for short-term use.
  • Scope and Lifetime of Variables: Understand where variables are accessible to avoid unintended errors and conflicts.
  • Importing Modules: Facilitates code reuse and access to a wide range of functionalities, both built-in and third-party.
  • Creating and Using Custom Modules: Allows for organization of code into separate files, making it easier to manage and reuse.

Data Structures in Python

Python offers a variety of built-in data structures that help manage and manipulate data efficiently. Let's explore the key data structures: Lists, Tuples, Sets, and Dictionaries.

1. Lists

Lists are ordered collections of items that can be changed (mutable). Lists can contain elements of different data types.

Common Operations:

  • Indexing: Access elements by their position in the list.
  • Slicing: Extract a portion of the list.
  • Adding Elements: Add new items to the list.
  • Removing Elements: Remove items from the list.

Example:

# Creating a list
fruits = ['apple', 'banana', 'cherry', 'date']

# Indexing
print(fruits[1])  # Output: banana

# Slicing
print(fruits[1:3])  # Output: ['banana', 'cherry']

# Adding Elements
fruits.append('elderberry')
print(fruits)  # Output: ['apple', 'banana', 'cherry', 'date', 'elderberry']

# Removing Elements
fruits.remove('date')
print(fruits)  # Output: ['apple', 'banana', 'cherry', 'elderberry']

Uses:

  • Maintaining an ordered collection of items.
  • Storing multiple items of different types.

2. Tuples

Tuples are ordered collections of items that cannot be changed (immutable). Tuples are similar to lists but have a fixed size.

Common Operations:

  • Accessing Elements: Retrieve items using indices.
  • Modifying Elements: Not allowed as tuples are immutable.

Example:

# Creating a tuple
dimensions = (20, 40, 60)

# Accessing Elements
print(dimensions[1])  # Output: 40

# Tuples are immutable, so modifying elements is not allowed
# dimensions[1] = 50  # This will raise a TypeError

Uses:

  • Storing fixed collections of items.
  • Used for immutable data that should not change after creation.

3. Sets

Sets are unordered collections of unique items. Sets are mutable but do not allow duplicate values.

Common Operations:

  • Union: Combine elements from multiple sets.
  • Intersection: Find common elements between sets.
  • Difference: Find elements present in one set but not in another.

Example:

# Creating sets
set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}

# Union
print(set1.union(set2))  # Output: {1, 2, 3, 4, 5, 6}

# Intersection
print(set1.intersection(set2))  # Output: {3, 4}

# Difference
print(set1.difference(set2))  # Output: {1, 2}

Uses:

  • Managing collections of unique items.
  • Performing set operations like union, intersection, and difference.

4. Dictionaries

Dictionaries are unordered collections of key-value pairs. Each key must be unique, and values can be of any data type.

Common Operations:

  • Accessing Values: Retrieve values using keys.
  • Adding Key-Value Pairs: Insert new items.
  • Removing Key-Value Pairs: Delete items by key.

Example:

# Creating a dictionary
person = {'name': 'Alice', 'age': 30, 'city': 'New York'}

# Accessing Values
print(person['name'])  # Output: Alice

# Adding Key-Value Pairs
person['occupation'] = 'Engineer'
print(person)  # Output: {'name': 'Alice', 'age': 30, 'city': 'New York', 'occupation': 'Engineer'}

# Removing Key-Value Pairs
del person['age']
print(person)  # Output: {'name': 'Alice', 'city': 'New York', 'occupation': 'Engineer'}

Uses:

  • Storing data as key-value pairs for quick lookups.
  • Implementing associative arrays or hash tables.

5. Collections Module

The collections module provides specialized data structures beyond the built-in types.

Common Data Structures:

  • namedtuple: Create tuple subclasses with named fields.
  • deque: Double-ended queue for fast appends and pops.
  • Counter: Count occurrences of elements.
  • OrderedDict: Dictionary that remembers the order of items.

Example:

from collections import namedtuple, deque, Counter, OrderedDict

# namedtuple
Person = namedtuple('Person', 'name age')
alice = Person(name='Alice', age=30)
print(alice.name)  # Output: Alice

# deque
dq = deque([1, 2, 3])
dq.append(4)
dq.appendleft(0)
print(dq)  # Output: deque([0, 1, 2, 3, 4])

# Counter
elements = ['apple', 'banana', 'apple', 'orange']
counter = Counter(elements)
print(counter)  # Output: Counter({'apple': 2, 'banana': 1, 'orange': 1})

# OrderedDict
ordered_dict = OrderedDict()
ordered_dict['first'] = 1
ordered_dict['second'] = 2
print(ordered_dict)  # Output: OrderedDict([('first', 1), ('second', 2)])

Uses:

  • Creating custom data structures.
  • Efficiently managing data with special requirements.

Reading from and Writing to Files in Python

1.1 Text Files

Reading from a Text File

# Reading from a text file
with open('example.txt', 'r') as file:
    content = file.read()
    print(content)

Explanation:

  • open('example.txt', 'r') opens the file example.txt in read mode.
  • file.read() reads the entire content of the file into a string.
  • with ensures the file is properly closed after its suite finishes.

Output: If example.txt contains:

Hello, world!
This is a sample text file.

The output will be:

Hello, world!
This is a sample text file.

Writing to a Text File

# Writing to a text file
with open('example.txt', 'w') as file:
    file.write('Hello, world!\nThis is a new line of text.')

Explanation:

  • open('example.txt', 'w') opens the file in write mode, creating it if it does not exist or truncating it if it does.
  • file.write() writes the string to the file.

Output: After running the above code, example.txt will contain:

Hello, world!
This is a new line of text.

1.2 Binary Files

Reading from a Binary File

# Reading from a binary file
with open('example.bin', 'rb') as file:
    content = file.read()
    print(content)

Explanation:

  • open('example.bin', 'rb') opens the file example.bin in binary read mode.
  • file.read() reads the entire content of the binary file.

Writing to a Binary File

# Writing to a binary file
with open('example.bin', 'wb') as file:
    file.write(b'\x00\x01\x02\x03')

Explanation:

  • open('example.bin', 'wb') opens the file in binary write mode.
  • file.write() writes bytes to the file.

Output: The file example.bin will contain binary data, which might not be human-readable but will be correctly formatted for binary applications.

1.3 Using with Statement

# Using 'with' statement for file handling
with open('example.txt', 'r') as file:
    content = file.read()
    print(content)

Explanation:

  • The with statement ensures that the file is properly closed after its suite finishes, making code more concise and readable.

1.4 File Methods

Common File Methods:

  • open(): Opens a file.
  • read(size): Reads up to size bytes. If size is not specified, it reads the entire file.
  • write(string): Writes string to the file.
  • close(): Closes the file.

Example:

# Example showing various file methods
with open('example.txt', 'w') as file:
    file.write('Hello, world!\n')
    file.write('Writing more text.')

with open('example.txt', 'r') as file:
    print(file.read())

Output:

Hello, world!
Writing more text.

2. Handling CSV and JSON Files

2.1 Handling CSV Files

Python provides the csv module to handle CSV files.

Reading CSV Files:

import csv

# Reading from a CSV file
with open('example.csv', 'r') as file:
    reader = csv.reader(file)
    for row in reader:
        print(row)

Explanation:

  • csv.reader(file) reads the CSV file and returns an iterator.
  • Each row is a list of strings from the CSV file.

Example CSV File (example.csv):

Name,Age,Occupation
Alice,30,Engineer
Bob,25,Artist

Output:

['Name', 'Age', 'Occupation']
['Alice', '30', 'Engineer']
['Bob', '25', 'Artist']

Writing to CSV Files:

import csv

# Writing to a CSV file
with open('example.csv', 'w', newline='') as file:
    writer = csv.writer(file)
    writer.writerow(['Name', 'Age', 'Occupation'])
    writer.writerow(['Alice', '30', 'Engineer'])
    writer.writerow(['Bob', '25', 'Artist'])

Explanation:

  • csv.writer(file) creates a writer object.
  • writer.writerow() writes a row to the CSV file.

2.2 Handling JSON Files

Python provides the json module to handle JSON files.

Reading JSON Files:

import json

# Reading from a JSON file
with open('example.json', 'r') as file:
    data = json.load(file)
    print(data)

Explanation:

  • json.load(file) parses the JSON file and returns a Python dictionary.

Example JSON File (example.json):

{
    "name": "Alice",
    "age": 30,
    "occupation": "Engineer"
}

Output:

{'name': 'Alice', 'age': 30, 'occupation': 'Engineer'}

Writing to JSON Files:

import json

# Writing to a JSON file
data = {
    "name": "Alice",
    "age": 30,
    "occupation": "Engineer"

with open('example.json', 'w') as file:

Exception Handling in Python

1. try, except, else, finally Blocks

try Block:

The try block contains code that might raise an exception. If an exception occurs, the rest of the try block is skipped, and the except block is executed.

except Block:

The except block catches and handles the exception raised by the try block. You can specify the type of exception to catch or use a generic except block to catch all exceptions.

else Block:

The else block is optional and executes if no exceptions occur in the try block. It is used to run code that should execute only if the try block succeeds.

finally Block:

The finally block is optional and always executes, regardless of whether an exception was raised or not. It is often used for cleanup actions, such as closing files or releasing resources.

Example:

def divide_numbers(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Error: Division by zero is not allowed.")
    except TypeError:
        print("Error: Invalid input type. Both arguments must be numbers.")
    else:
        print(f"Result: {result}")
    finally:
        print("Execution completed.")

# Valid division
divide_numbers(10, 2)

# Division by zero
divide_numbers(10, 0)

# Invalid input type
divide_numbers(10, 'a')

Output:

Result: 5.0
Execution completed.
Error: Division by zero is not allowed.
Execution completed.
Error: Invalid input type. Both arguments must be numbers.
Execution completed.

Explanation:

  • In the first call, 10 / 2 is successful, so the else block executes, and "Result: 5.0" is printed. The finally block always executes, printing "Execution completed."
  • In the second call, a ZeroDivisionError occurs, so the except block for ZeroDivisionError handles the exception. The finally block then executes.
  • In the third call, a TypeError occurs due to invalid input. The except block for TypeError handles the exception, followed by the finally block.

2. Raising Exceptions

Sometimes you may need to raise exceptions intentionally using the raise keyword. This is useful for implementing custom error conditions.

Example:

def check_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative.")
    elif age < 18:
        raise ValueError("Age must be at least 18.")
    else:
        print("Age is valid.")

try:
    check_age(25)  # Valid age
    check_age(-5)  # Invalid age
except ValueError as e:
    print(f"Error: {e}")

Output:

Age is valid.
Error: Age cannot be negative.

Explanation:

  • The first call to check_age(25) is valid, so it prints "Age is valid."
  • The second call raises a ValueError, which is caught in the except block, printing "Error: Age cannot be negative."

3. Custom Exceptions

You can define your own exception classes by inheriting from the built-in Exception class. Custom exceptions allow you to create more specific and meaningful error messages.

Example:

class CustomError(Exception):
    def __init__(self, message):
        self.message = message
        super().__init__(self.message)

def process_data(data):
    if not isinstance(data, list):
        raise CustomError("Data must be a list.")
    if len(data) == 0:
        raise CustomError("Data list cannot be empty.")
    print("Processing data:", data)

try:
    process_data([1, 2, 3])  # Valid data
    process_data("string")  # Invalid data
    process_data([])        # Empty list
except CustomError as e:
    print(f"CustomError: {e}")

Output:

Processing data: [1, 2, 3]
CustomError: Data must be a list.
CustomError: Data list cannot be empty.

Explanation:

  • The first call to process_data([1, 2, 3]) is valid and prints the data.
  • The second call raises a CustomError because the data is not a list.
  • The third call raises another CustomError because the list is empty.

Summary:

  • try: Block where code that may raise an exception is placed.
  • except: Block to handle specific exceptions.
  • else: Block that executes if no exceptions occur.
  • finally: Block that always executes for cleanup.
  • Raising Exceptions: Use raise to intentionally raise exceptions.
  • Custom Exceptions: Define your own exception classes for specific error handling.

Classes and Objects in Python

Concept:

Class: A blueprint for creating objects. It encapsulates data for the object and methods to manipulate that data.

Object: An instance of a class. It contains data and methods defined in the class.

Example:

class Dog:
    # Constructor
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Method
    def bark(self):
        return f"{self.name} says Woof!"

# Create an object of the class
my_dog = Dog(name="Buddy", age=3)

# Accessing attributes and methods
print(f"My dog's name is {my_dog.name}.")  # Output: My dog's name is Buddy.
print(f"My dog is {my_dog.age} years old.")  # Output: My dog is 3 years old.
print(my_dog.bark())  # Output: Buddy says Woof!

Attributes and Methods

Concept:

Attributes: Variables bound to the instance of a class.

Methods: Functions defined inside a class that operate on attributes of the class.

Example:

class Car:
    def __init__(self, make, model):
        self.make = make
        self.model = model

    def display_info(self):
        return f"Car: {self.make} {self.model}"

# Create an object
my_car = Car(make="Toyota", model="Corolla")

# Accessing attributes and methods
print(my_car.display_info())  # Output: Car: Toyota Corolla

Inheritance

Concept: Inheritance allows a class to inherit attributes and methods from another class.

Example:

# Base class
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound."

# Derived class
class Cat(Animal):
    def speak(self):
        return f"{self.name} says Meow!"

# Create an object of the derived class
my_cat = Cat(name="Whiskers")

print(my_cat.speak())  # Output: Whiskers says Meow!

Polymorphism

Concept: Polymorphism allows methods to do different things based on the object it is acting upon, even if they share the same name.

Example:

class Bird:
    def speak(self):
        return "Tweet"

class Dog:
    def speak(self):
        return "Woof"

# Function demonstrating polymorphism
def animal_sound(animal):
    print(animal.speak())

# Create objects
bird = Bird()
dog = Dog()

animal_sound(bird)  # Output: Tweet
animal_sound(dog)   # Output: Woof

Encapsulation and Abstraction

Concept:

Encapsulation: Hides the internal state of an object and only exposes a controlled interface.

Abstraction: Provides a simplified view of complex systems, showing only relevant details.

Example:

class BankAccount:
    def __init__(self, account_number, balance):
        self.__account_number = account_number  # Private attribute
        self.__balance = balance  # Private attribute

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
        else:
            raise ValueError("Deposit amount must be positive.")

    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
        else:
            raise ValueError("Insufficient funds or invalid amount.")

    def get_balance(self):
        return self.__balance

# Create an object
account = BankAccount("123456", 1000)

# Access methods to interact with the private data
account.deposit(500)
print(account.get_balance())  # Output: 1500
account.withdraw(200)
print(account.get_balance())  # Output: 1300

Method Overloading and Overriding

Concept:

Method Overloading: Python does not support method overloading as it does not allow multiple methods with the same name but different parameters. However, you can achieve similar functionality using default arguments.

Method Overriding: A subclass can provide a specific implementation of a method that is already defined in its superclass.

Example:

class Parent:
    def greet(self, name="Guest"):
        return f"Hello, {name}!"

class Child(Parent):
    def greet(self, name="Child"):
        return f"Hi, {name}!"

# Create objects
parent = Parent()
child = Child()

print(parent.greet())  # Output: Hello, Guest!
print(parent.greet("Alice"))  # Output: Hello, Alice!

print(child.greet())  # Output: Hi, Child!
print(child.greet("Bob"))  # Output: Hi, Bob!

Explanation:

  • Classes and Objects: Classes are blueprints for creating objects. Objects are instances of classes that have attributes and methods.
  • Attributes and Methods: Attributes are the properties of a class, while methods are functions that operate on these attributes.
  • Inheritance: Allows a class to inherit attributes and methods from another class, enabling code reuse.
  • Polymorphism: Allows different classes to be treated through the same interface, with each class implementing methods in its own way.
  • Encapsulation and Abstraction: Encapsulation hides the internal details of an object, while abstraction provides a simplified interface to interact with objects.
  • Method Overloading and Overriding: Overloading is not natively supported in Python, but overriding allows subclasses to provide specific implementations of methods defined in parent classes.

Python Programming Concepts

1. Decorators

Definition: Decorators are a way to modify or extend the behavior of functions or methods without changing their actual code. They are used to add functionality to existing code in a clean and readable way.

How They Work: A decorator is a function that takes another function and extends its behavior.

def my_decorator(func):
    def wrapper():
        print("Something is happening before the function is called.")
        func()
        print("Something is happening after the function is called.")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()

Output:

Something is happening before the function is called.
Hello!
Something is happening after the function is called.

Explanation:

  • my_decorator is a decorator function.
  • It wraps the say_hello function, adding behavior before and after its execution.
  • @my_decorator is a syntactic sugar for say_hello = my_decorator(say_hello).

Uses:

  • Logging
  • Access control
  • Caching
  • Validation

2. Generators

Definition: Generators are a type of iterable that generate values one at a time and only when needed. They allow for lazy evaluation and are more memory efficient than traditional functions that return lists.

How They Work: Generators use the yield keyword to produce a series of values over time, pausing and resuming their state between yields.

def count_up_to(max):
    count = 1
    while count <= max:
        yield count
        count += 1

for number in count_up_to(5):
    print(number)

Output:

1
2
3
4
5

Explanation:

  • count_up_to is a generator function.
  • It yields values one at a time, allowing iteration over the values produced.

Uses:

  • Iterating over large data sets
  • Streaming data
  • Implementing custom iteration behavior

3. Context Managers

Definition: Context managers are used for resource management, such as opening files or managing connections. They ensure that resources are properly cleaned up after use, even if errors occur.

How They Work: Context managers use the with statement to ensure that resources are acquired and released properly.

with open('example.txt', 'w') as file:
    file.write('Hello, world!')

Output:

A file named example.txt will be created with the content "Hello, world!".

Explanation:

  • The with statement automatically handles opening and closing the file.
  • The file is closed automatically when the block of code is exited.

Uses:

  • File operations
  • Database connections
  • Resource management

Custom Context Manager Example:

class MyContextManager:
    def __enter__(self):
        print("Entering the context")
        return self
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        print("Exiting the context")

with MyContextManager() as manager:
    print("Inside the context")

Output:

Entering the context
Inside the context
Exiting the context

Explanation:

  • __enter__ and __exit__ methods define the behavior of the context manager.
  • Resources are acquired and released in a controlled manner.

4. Iterators

Definition: Iterators are objects that represent a stream of data. They implement two methods: __iter__() and __next__(), allowing for iteration over a sequence of values.

How They Work: Iterators provide a way to traverse a container, such as a list or dictionary, without exposing its underlying structure.

class Count:
    def __init__(self, low, high):
        self.current = low
        self.high = high
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.current > self.high:
            raise StopIteration
        else:
            self.current += 1
            return self.current - 1

counter = Count(1, 5)
for num in counter:
    print(num)

Output:

1
2
3
4
5

Explanation:

  • Count class implements the iterator protocol with __iter__ and __next__.
  • Iteration proceeds through the values defined in the range.

Uses:

  • Custom iteration logic
  • Implementing iterable objects
  • Traversing large data sets

5. Metaclasses

Definition: Metaclasses are classes of classes. They define how classes themselves are created and behave. In Python, everything is an object, including classes.

How They Work: A metaclass is a class that defines the behavior of other classes.

class Meta(type):
    def __new__(cls, name, bases, dct):
        print(f"Creating class {name}")
        return super().__new__(cls, name, bases, dct)

class MyClass(metaclass=Meta):
    pass

Output:

Creating class MyClass

Explanation:

  • Meta is a metaclass that prints a message when a class is created.
  • MyClass uses Meta as its metaclass.

Uses:

  • Customizing class creation
  • Implementing class-level behavior
  • Enforcing design patterns

6. Coroutines

Definition: Coroutines are a special type of generator used for cooperative multitasking. They allow functions to pause and resume their execution, making them suitable for asynchronous programming.

How They Work: Coroutines use async def to define asynchronous functions and await to pause execution until the awaited result is ready.

import asyncio

async def say_hello():
    print("Hello")
    await asyncio.sleep(1)
    print("World")

asyncio.run(say_hello())

Output:

Hello
(After 1 second)
World

Explanation:

  • say_hello is an asynchronous coroutine function.
  • await asyncio.sleep(1) pauses execution for 1 second before printing "World".

Uses:

  • Asynchronous programming
  • Concurrency and parallelism
  • Handling I/O-bound operations
