Introduction to React

Overview of React

React is a popular open-source JavaScript library used for building user interfaces, particularly single-page applications where data changes over time. Developed and maintained by Facebook, React allows developers to create large web applications that can update and render efficiently in response to data changes. React’s key feature is the ability to break down complex UI into simpler components, which can be reused throughout the application. It emphasizes declarative programming, where the UI is described as a function of the application state.

History and Evolution

React was first developed by Jordan Walke, a software engineer at Facebook, and was initially deployed on Facebook’s newsfeed in 2011. It was later open-sourced at the JSConf US conference in May 2013. Since its release, React has undergone significant evolution:

  • 2013: Initial release of React (React 0.3.0).
  • 2014: Introduction of the React component lifecycle and improved server-side rendering.
  • 2015: Release of React 0.14, separating React and ReactDOM, enabling the use of React Native.
  • 2016: Introduction of the Context API and React 15 with a focus on more accurate and consistent handling of components.
  • 2017: Release of React 16 (codenamed "Fiber"), which brought major improvements in performance, error boundaries, and introduced portals.
  • 2018: Introduction of React Hooks, providing a new way to use state and other React features without writing a class.
  • 2020: React 17 ("No New Features") focused on making it easier to upgrade React itself, enabling gradual upgrades.
  • 2022: Introduction of React 18, featuring concurrent rendering, automatic batching, and more.

React’s evolution has focused on performance improvements, better developer experience, and enhancing the capability of building scalable and maintainable applications.

Advantages of Using React

React offers several advantages that make it a preferred choice for developers:

  • Component-Based Architecture: React’s component-based structure allows developers to build encapsulated components that manage their state and compose them to make complex UIs. This modularity improves code maintainability and reusability.
  • Virtual DOM: React uses a Virtual DOM, which is a lightweight copy of the real DOM. When the state of an object changes, React updates only the Virtual DOM first. Then, it compares the new Virtual DOM with a pre-updated version and updates only the parts of the actual DOM that have changed, resulting in better performance.
  • Declarative UI: React makes it easy to design interactive UIs with its declarative approach. Developers simply describe how the UI should look based on the application’s state, and React takes care of updating the UI when the state changes.
  • Unidirectional Data Flow: React’s unidirectional data flow ensures that data moves in a single direction, making it easier to debug and understand how data changes affect the application’s state and UI.
  • Rich Ecosystem and Community Support: React has a vast ecosystem of tools, libraries, and a large community that contributes to its growth and provides support for developers.
  • React Native: React can be used to build mobile applications using React Native, allowing developers to use the same design principles and components to create cross-platform apps for iOS and Android.
  • SEO-Friendly: With React's server-side rendering capabilities, developers can create applications that are more SEO-friendly, allowing better indexing by search engines.

React vs. Other Front-End Frameworks

React is often compared with other popular front-end frameworks such as Angular, Vue.js, and Svelte. Here’s how React stands out:

  • React vs. Angular:
    • Flexibility: React is a library focused on building UI components, giving developers the flexibility to integrate other libraries as needed. Angular is a full-fledged framework with a more opinionated structure, providing everything out of the box.
    • Learning Curve: React’s learning curve is generally considered less steep than Angular’s, as it only requires knowledge of JavaScript and a few core concepts like components, state, and props.
    • Performance: Both React and Angular offer good performance, but React’s Virtual DOM and the ability to fine-tune updates give it an edge in certain scenarios.
  • React vs. Vue.js:
    • Adoption and Ecosystem: React has broader adoption and a larger ecosystem compared to Vue.js, which can be advantageous for finding libraries, tools, and support.
    • Flexibility: Vue.js also offers a component-based architecture and is similar to React in many ways. However, React’s unidirectional data flow contrasts with Vue’s two-way data binding, giving React more predictable state management.
    • Learning Curve: Vue.js is often praised for its simplicity and ease of integration with existing projects, making it a good choice for smaller applications or those transitioning from a different technology.
  • React vs. Svelte:
    • Performance: Svelte takes a different approach by compiling components into highly efficient imperative code that manipulates the DOM directly, often resulting in better performance than React’s Virtual DOM.
    • Popularity: React has a much larger community and ecosystem compared to Svelte, making it easier to find resources, libraries, and developers familiar with it.
    • Development Experience: Svelte offers a more straightforward development experience by eliminating the need for a Virtual DOM and using more concise syntax. However, React’s ecosystem and maturity make it a more reliable choice for large-scale applications.

React’s simplicity, flexibility, and strong community support make it a powerful tool for modern web development, and its continued evolution ensures it remains relevant and competitive in the rapidly changing front-end landscape.

Creating react app

Open your terminal in the directory you would like to create your application. Run this command to create a React application named my-react-app:

npx create-react-app my-react-app

OR, you can directly make your application without specifying a name, like this:

npx create-react-app .

In this case, all files will be kept in the current directory.

Note :When choosing folder name, make sure there are no spaces or capital letters because of npm naming restrictions.

Once base application is created, if folder specified you just have to enter the folder. You can use this command to enter:

cd directory-name

Then just start up the application with this command:

npm start

and you are good to go!

What is ES6?

ES6 stands for ECMAScript 6. ECMAScript is a JavaScript standard intended to ensure a common language across different browsers. ES6 is the 6th version of ECMAScript.


Why ES6? / Features of ES6 / Upgrades in ES6

React uses ES6 and all of these new features will make your coding experience in react much much better. You will be able to do things with much more ease and in very less lines! Features like:

  • Arrow Functions:
    • const hello = () => {
    return "Hello World!";
}

    Or

    const hello = () => "Hello World!";
  • .map():.map can be used for alot of things, one of it's use case is, we can make any number of cards through loop and just put it in jsx, like this:
    • const data = ['title1', 'title2', 'title3'];
let cards = data.map((item) => {item})
  • Destructuring:
    Old way:
    • const languages = ['JS', 'Python', 'Java'];
const js = languages[0]
const python = languages[1]
const java = languages[2]

    New Ways:

    const languages = ['JS', 'Python', 'Java'];
const [ js, python, java ] = languages
  • Ternary Operator: With this, you can write if/else conditions in one line. It's syntax is fairly simple like this:
    • condition ? <expression if true> : <expression if false>

    Example:

    let loading = false; 
const data = loading ? <div>Loading...</div> : <div>Data</div>
  • Spread Operator: 
    • const languages = ['JS', 'Python', 'Java'];
const morelanguages = ['C', 'C++', 'C#']

const allLanguages = [...languages, ...morelanguages]

    Output:

    ["JS","Python","Java","C","C++","C#"]

    React Components

    There are two types of components:

    • Class Based Components
    • Function Based Components

    Class Based Components

    Before making class based component we need to inherit functions from React.Component and this can be done with extends, like this:

    class Cat extends React.Component {
        render() {
                return <h1>Meow</h1>;
            }
}

    it also requires a render method which returns HTML.


    Function Based Components

    In function it's simpler, we just need to return the HTML, like this:

    function Cat() {
    return <h1>Meow>/h1>;
}
    Note:Component's name must start with uppercase letter.

    Rendering a Component

    We made a component, now we want to render/use it. Syntax for using a component is: 

    <ComponentName />

    Components in Files

    To have less mess inside main file(with all the components in the same file) and to resuse components on different pages, we have to make them separately. So that we can just import them in any file and use them!

    For that we will just make a new file called Cat.js, make class or function based component there and export default that class/function! Like this: 

    function Cat() {
    return <h1>Meow</h1>;
}
                      
export default Cat;
    Note:File name must start with uppercase letter.

    Props

    As mentioned earlier, we can import the same component in different files and use it, but maybe in different files some changes in the component is needed. For that, we can use props! Like this:

    Component: 
    function Cat(props) {
    return <h1>Meow's color is {props.color}</h1>;
}

    Main file 
    <Cat color="purple" />

    Class Based Components

    Before making class based component we need to inherit functions from React.Component and this can be done with extends, like this:

    class Cat extends React.Component {
    render() {
        return <h1>Meow</h1>;
    }
}

    it also requires a render method which returns HTML.

    Note: Component's name must start with uppercase letter.

    Component Constructor

    Constructor gets called when the component is initiated. This is where you initiate the component's properties. In React we have states which update on page without reload. Constructor properties are kept in state.

    We also need to add super() statement, which executes the parent component's constructor and component gets access to all the functions of the parent component, like this:

    class Cat extends React.Component {
    constructor() {
        super();
        this.state = { color: "orange" };
    }
    render() {
        return <h1>Meow's color is {this.state.color}>/h1>;
    }
}

    React Props

    Props are arguments passed to React components via HTML attributes. Example:

    Component: 
    function Cat(props) {
    return <h1>Meow's color is {props.color}</h1>;
}

        <Cat color="purple" />
